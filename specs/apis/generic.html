Mode generique pour toute API REST retournant du JSON. C'est le mode de repli (fallback) quand l'URL ne correspond a aucun fournisseur connu.

Tout cote client

Le mode generique ne connait pas la structure de l'API cible. Il recupere les donnees via dsfr-data-source et toutes les transformations (tri, filtrage, agregation, facettes, recherche) sont effectuees cote client par dsfr-data-query.

Capacites serveur

Capacite Support Detail
Fetch serveur Non Pas de pagination automatique
Facettes serveur Non Facettes calculees cote client
Recherche serveur Non Recherche effectuee cote client
Group-by serveur Non Group-by effectue cote client
Tri serveur Non Tri effectue cote client
Agregation serveur Non Agregation effectuee cote client

Pagination

Aucune pagination automatique. dsfr-data recupere l'URL telle quelle et traite la reponse JSON comme un tableau de donnees. Si votre API utilise de la pagination, vous devez la gerer manuellement dans l'URL ou utiliser un des fournisseurs specialises.

Structure de reponse

ParametreValeur
Chemin des donnees(racine du JSON)
Chemin du totalNon disponible
Donnees imbriqueesNon
Necessite flattenNon

La reponse JSON doit etre un tableau d'objets a la racine, ou vous pouvez utiliser l'attribut data-path sur dsfr-data-source pour specifier le chemin vers le tableau de donnees dans la reponse.

Syntaxe de requete

Le mode generique n'envoie aucun parametre de requete a l'API. Toutes les transformations sont effectuees cote client par dsfr-data-query avec la syntaxe colon generique (field:operator:value).

Facettes

ParametreValeur
Mode par defautclient
Endpoint serveurNon disponible

Authentification

Type : aucune. Le mode generique ne gere pas l'authentification automatiquement. Si votre API necessite une cle, passez-la directement dans l'URL ou configurez un proxy.

Detection automatique

Le mode generique est le fallback : il est selectionne automatiquement quand l'URL ne correspond a aucun des autres fournisseurs (OpenDataSoft, Tabular, Grist).

Exemple d'utilisation

API JSON simple

<dsfr-data-source
  id="src"
  url="https://mon-api.example.com/data.json">
</dsfr-data-source>

<dsfr-data-chart
  source="src"
  type="bar"
  name-field="categorie"
  value-field="valeur">
</dsfr-data-chart>

API avec chemin de donnees

<!-- Si la reponse JSON est { "results": { "items": [...] } } -->
<dsfr-data-source
  id="src"
  url="https://mon-api.example.com/endpoint"
  data-path="results.items">
</dsfr-data-source>

<dsfr-data-query
  source="src"
  group-by="type"
  aggregate="montant:sum">
</dsfr-data-query>

<dsfr-data-chart
  source="src"
  type="pie"
  name-field="type"
  value-field="montant">
</dsfr-data-chart>

Avec normalize pour des donnees imbriquees

<dsfr-data-source
  id="src"
  url="https://mon-api.example.com/nested-data.json">
</dsfr-data-source>

<dsfr-data-normalize
  source="src"
  id="flat"
  flatten="data">
</dsfr-data-normalize>

<dsfr-data-list
  source="flat"
  columns="nom,valeur,date"
  page-size="20">
</dsfr-data-list>
