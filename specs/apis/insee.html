API Melodi de l'INSEE, donnant acces a plus de 95 jeux de données statistiques (population, PIB, emploi, etc.) du catalogue de données de l'INSEE.

Données statistiques dimensionnelles

L'API Melodi utilise un modele dimensionnel : chaque observation est decrite par des dimensions (GEO, TIME_PERIOD, FREQ...), une mesure (OBS_VALUE) et des attributs optionnels. Le filtrage se fait par egalite sur les dimensions via des parametres de requête. L'adapter aplatit automatiquement les observations en objets plats exploitables par les composants dsfr-data.

CORS actif, pas de proxy

L'API INSEE supporte nativement CORS. Aucun proxy n'est necessaire. Acces anonyme, limite a 30 requêtes par minute.

Filtrage par dimensions

L'API Melodi filtre par egalite sur les dimensions via des parametres de requête. Chaque jeu de données a ses propres dimensions (GEO, TIME_PERIOD, FREQ, etc.).

Syntaxe API native

?TIME_PERIOD=2023&GEO=FRANCE-F&POPREF_MEASURE=PMUN

Multi-valeurs : repeter le parametre (?POPREF_MEASURE=PMUN&POPREF_MEASURE=PTOT).

Syntaxe colon (dsfr-data-source)

L'adapter convertit la syntaxe colon standard en parametres INSEE :

Attribut whereParamètre URL généré
TIME_PERIOD:eq:2023?TIME_PERIOD=2023
GEO:in:FRANCE-F|FRANCE-M?GEO=FRANCE-F&GEO=FRANCE-M
TIME_PERIOD:eq:2023, GEO:eq:FRANCE-F?TIME_PERIOD=2023&GEO=FRANCE-F

Seuls les operateurs eq et in sont geres cote serveur. Les operateurs avances (gt, lt, contains...) sont appliques cote client par dsfr-data-query.

Structure de reponse

Les observations INSEE ont une structure dimensionnelle imbriquee, automatiquement aplatie par l'adapter :

ParametreValeur
Chemin des donnéesobservations
Chemin du totalpaging.count
Données imbriqueesOui : dimensions, measures, attributes
Aplatissement autoOui (par l'adapter, pas besoin de dsfr-data-normalize)

Format brut de l'API

{
  "observations": [{
    "dimensions": { "GEO": "2025-FRANCE-F", "FREQ": "A", "TIME_PERIOD": "2023" },
    "measures": { "OBS_VALUE_NIVEAU": { "value": 68094280.0 } },
    "attributes": { "OBS_STATUS": "A" }
  }],
  "paging": { "count": 35355, "next": "...&page=2" }
}

Apres aplatissement (exploitable par dsfr-data-query)

{
  "GEO": "2025-FRANCE-F",
  "FREQ": "A",
  "TIME_PERIOD": "2023",
  "OBS_VALUE": 68094280.0,
  "OBS_STATUS": "A"
}

La mesure OBS_VALUE_NIVEAU est renommee OBS_VALUE (le suffixe _NIVEAU est supprime). D'autres mesures suivent le même pattern.

Catalogue des jeux de données

L'endpoint /catalog/all liste les 95+ jeux de données disponibles. Chaque entree contient l'identifiant, le titre, la description, le nombre d'observations et la liste des dimensions (ordreComposants).

Authentification

Type : aucune. L'API Melodi est publique et ne necessite pas d'authentification. Le debit est limite a 30 requêtes par minute en mode anonyme.

Detection automatique

dsfr-data détecté automatiquement une source INSEE si l'URL contient :

  • melodi/data/{datasetId}

Base URL : https://api.insee.fr/melodi

Exemples d'utilisation

Population municipale de France

<dsfr-data-source
  id="src"
  api-type="insee"
  base-url="https://api.insee.fr/melodi"
  dataset-id="DS_POPULATIONS_REFERENCE"
  where="POPREF_MEASURE:eq:PMUN, TIME_PERIOD:eq:2023, GEO:eq:FRANCE-F">
</dsfr-data-source>

<dsfr-data-kpi
  source="src"
  valeur="OBS_VALUE"
  label="Population municipale 2023"
  format="nombre">
</dsfr-data-kpi>

Populations par departement avec graphique

<dsfr-data-source
  id="src"
  api-type="insee"
  base-url="https://api.insee.fr/melodi"
  dataset-id="DS_POPULATIONS_REFERENCE"
  where="POPREF_MEASURE:eq:PMUN, TIME_PERIOD:eq:2023"
  limit="500">
</dsfr-data-source>

<dsfr-data-query
  source="src"
  filter="GEO:contains:DEP"
  order-by="OBS_VALUE:desc">
</dsfr-data-query>

<dsfr-data-chart
  source="src"
  type="bar"
  name-field="GEO"
  value-field="OBS_VALUE">
</dsfr-data-chart>

Decoder les codes de dimension avec dsfr-data-normalize

Les dimensions INSEE utilisent des codes (AGE: "Y30T39", PCS: "3"). L'attribut replace-fields de <dsfr-data-normalize> permet de les remplacer par des libelles lisibles, sans affecter les autres champs qui pourraient contenir les mêmes valeurs. 

<dsfr-data-source
  id="raw"
  api-type="insee"
  base-url="https://api.insee.fr/melodi"
  dataset-id="DS_POPULATIONS_REFERENCE"
  where="POPREF_MEASURE:eq:PMUN, TIME_PERIOD:eq:2023"
  limit="500">
</dsfr-data-source>

<dsfr-data-normalize id="decoded" source="raw"
  replace-fields="AGE:Y30T39:30-39 ans | AGE:Y_LT30:Moins de 30 ans | PCS:3:Cadres | PCS:5:Employes"
  replace="N/A:">
</dsfr-data-normalize>

<dsfr-data-chart
  source="decoded"
  type="bar"
  name-field="AGE"
  value-field="OBS_VALUE">
</dsfr-data-chart>

Datalist avec pagination serveur

<dsfr-data-source
