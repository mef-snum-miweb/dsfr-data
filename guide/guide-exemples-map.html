dsfr-data-map

Carte interactive multi-couches avec fonds de carte souverains (IGN Geoplateforme). Deux composants : dsfr-data-map (conteneur) et dsfr-data-map-layer (couche de donnees).

dsfr-data-map ne consomme pas de donnees. Ce sont les dsfr-data-map-layer enfants qui utilisent le SourceSubscriberMixin. Cela permet le multi-source naturellement : chaque couche peut avoir sa propre source de donnees.

POI avec markers — Centres de controle technique

Affichage de POI depuis une API OpenDataSoft. Les markers utilisent les icones DSFR. Les coordonnees sont extraites via lat-field + lon-field.

Les 200 centres les moins chers (voitures, essence)

Chargement...

<dsfr-data-source id="ct" api-type="opendatasoft"
  base-url="https://data.economie.gouv.fr"
  dataset-id="prix-controle-technique"
  where="cat_vehicule_id=1 AND cat_energie_id=1"
  select="cct_denomination,cct_commune,nom_departement,latitude,longitude,prix_visite"
  order-by="prix_visite" limit="200">
</dsfr-data-source>

<dsfr-data-map center="46.6,2.3" zoom="6" tiles="ign-plan" height="500px" name="200 centres CT les moins chers">
  <dsfr-data-map-layer source="ct" type="marker"
    lat-field="latitude" lon-field="longitude"
    popup-fields="cct_denomination,cct_commune,nom_departement,prix_visite"
    tooltip-field="cct_denomination">
  </dsfr-data-map-layer>
</dsfr-data-map>

Clustering — Gros volume de POI

Pour les gros volumes (milliers de markers), activez le clustering avec cluster. La librairie leaflet.markercluster est chargee dynamiquement. Ici, 5000 centres de controle technique.

5000 centres avec clustering

Chargement...

<dsfr-data-source id="ct-cluster" api-type="opendatasoft"
  base-url="https://data.economie.gouv.fr"
  dataset-id="prix-controle-technique"
  where="cat_vehicule_id=1 AND cat_energie_id=1"
  select="cct_denomination,cct_commune,latitude,longitude,prix_visite"
  limit="5000">
</dsfr-data-source>

<dsfr-data-map center="46.6,2.3" zoom="6" tiles="osm" height="500px">
  <dsfr-data-map-layer source="ct-cluster" type="marker"
    lat-field="latitude" lon-field="longitude"
    popup-template="{cct_denomination}<br>{cct_commune} — {prix_visite} EUR"
    tooltip-field="cct_denomination"
    cluster cluster-radius="60">
  </dsfr-data-map-layer>
</dsfr-data-map>

Heatmap — Densite de centres

Le type heatmap affiche une carte de chaleur via leaflet.heat. L'attribut heat-field permet de ponderer les points par une valeur numerique. Les attributs heat-radius et heat-blur controlent le rendu.

Densite des centres de controle technique en Ile-de-France

Chargement...

<dsfr-data-source id="ct-idf" api-type="opendatasoft"
  base-url="https://data.economie.gouv.fr"
  dataset-id="prix-controle-technique"
  where="cat_vehicule_id=1 AND cat_energie_id=1 AND code_region=11"
  select="latitude,longitude,prix_visite"
  limit="1000">
</dsfr-data-source>

<dsfr-data-map center="48.86,2.35" zoom="10" tiles="ign-cadastre" height="500px">
  <dsfr-data-map-layer source="ct-idf" type="heatmap"
    lat-field="latitude" lon-field="longitude"
    heat-radius="20" heat-blur="15">
  </dsfr-data-map-layer>
</dsfr-data-map>

Panneau lateral au clic (dsfr-data-map-popup)

Le composant dsfr-data-map-popup definit un template HTML et un mode d'affichage. En mode panel-right, un panneau lateral s'ouvre au clic sur un POI et se met a jour a chaque nouveau clic. Le template utilise la syntaxe {{champ}}.

Panneau lateral avec template personnalise

Chargement...

<dsfr-data-source id="ct-panel" api-type="opendatasoft"
  base-url="https://data.economie.gouv.fr"
  dataset-id="prix-controle-technique"
  where="cat_vehicule_id=1 AND cat_energie_id=1 AND code_departement='75'"
  select="cct_denomination,cct_adresse,cct_commune,cct_code_postal,latitude,longitude,prix_visite,prix_contre_visite_mini,prix_contre_visite_maxi"
  limit="50">
</dsfr-data-source>

<dsfr-data-map center="48.86,2.35" zoom="12" tiles="ign-plan" height="500px" name="Controles techniques Paris">
  <dsfr-data-map-layer source="ct-panel" type="marker"
    lat-field="latitude" lon-field="longitude"
    tooltip-field="cct_denomination">
  </dsfr-data-map-layer>

  <dsfr-data-map-popup mode="panel-right" title-field="cct_denomination" width="380px">

Animation temporelle (timeline)

L'attribut time-field sur un layer active le decoupage temporel des donnees. Le composant dsfr-data-map-timeline ajoute les controles de lecture (play, pause, pas-a-pas, vitesse). Ici, l'evolution fictive du nombre de bornes de recharge electrique par region sur 4 trimestres.

  • time-bucket="none" : utilise les valeurs brutes (ici les trimestres)
  • time-mode="snapshot" : affiche uniquement le trimestre courant
  • time-mode="cumulative" : cumule les donnees jusqu'au trimestre courant

Zoom ranges et multi-couches

Les attributs min-zoom / max-zoom sur les layers permettent d'afficher differentes couches selon le niveau de zoom. Combinee avec bbox pour le chargement par viewport, cette technique permet la multi-resolution administrative. 

Zoom 1-9  : regions (13 features, chargees une fois)
Zoom 10+  : communes dans le viewport (bbox re-fetch)
Zoom 13+  : POI dans le viewport (bbox re-fetch + cluster)

Viewport-driven fetch (bbox)

L'attribut bbox sur un layer active le chargement par viewport. A chaque deplacement de carte (debounce 300ms par defaut), le layer envoie une commande dsfr-data-source-command avec un filtre geographique.

  • ODS : filtre server-side in_bbox(field, lat1, lon1, lat2, lon2)
  • Tabular, Grist, Generic : fallback client-side (filtre en memoire)
