Dashboard Ghibli - API generic

Tableau de bord filmographique du Studio Ghibli, construit a partir d'une API REST generique (JSON plat, sans authentification, CORS natif). Illustre l'usage du provider generic avec le pipeline : source URL directe, normalisation, queries client-side et visualisations DSFR.

Provider generic : L'API Ghibli retourne directement un tableau JSON plat (pas d'enveloppe, CORS natif). dsfr-data-source utilise une URL directe sans api-type ni transform. Les champs numeriques (rt_score, running_time, release_date) arrivent en string et sont convertis via dsfr-data-normalize numeric. Toutes les transformations (tri, filtre, group-by, agregation) sont effectuees client-side par dsfr-data-query.

Architecture du pipeline

dsfr-data-source "raw-films" (fetch URL directe, JSON plat, CORS natif) \-- dsfr-data-normalize "films" (trim + numeric="rt_score,running_time,release_date") |-- dsfr-data-query "by-score" (order-by rt_score:desc) | \-- dsfr-data-chart bar horizontal (scores RT) |-- dsfr-data-query "by-duration" (order-by running_time:desc) | \-- dsfr-data-chart bar horizontal (durees) |-- dsfr-data-query "by-director-count" (group-by director, count) | \-- dsfr-data-chart pie (nb films/realisateur) |-- dsfr-data-query "by-director-score" (group-by director, avg rt_score) | \-- dsfr-data-chart bar (score moyen/realisateur) |-- dsfr-data-query "recent-films" (where release_date >= 2000) | \-- dsfr-data-chart line x2 (score + duree dans le temps) |-- dsfr-data-kpi x2 (score RT moyen, duree moyenne) \-- dsfr-data-facets + dsfr-data-list (filmographie complete)

Source : Ghibli API (ghibliapi.vercel.app) — donnees publiques, CORS natif, aucune cle d'API
Provider : generic (URL directe, tableau JSON plat, pas de transform)
Pipeline 100% declaratif : dsfr-data-source (fetch URL) → dsfr-data-normalize (trim + numeric) → 6× dsfr-data-query client-side (group-by, aggregate, where, order-by) → dsfr-data-facets (filtres interactifs) + 2× dsfr-data-kpi + 6× dsfr-data-chart + dsfr-data-list + 6× dsfr-data-a11y 

<!-- 1. Fetch generique (URL directe, CORS natif) -->
<dsfr-data-source id="raw-films"
  url="https://ghibliapi.vercel.app/films">
</dsfr-data-source>

<!-- 2. Normalisation : conversion numerique des champs string -->
<dsfr-data-normalize id="films" source="raw-films"
  trim
  numeric="rt_score,running_time,release_date">
</dsfr-data-normalize>

<!-- 3. Queries derivees (tout client-side) -->

<!-- Films tries par score RT decroissant -->
<dsfr-data-query id="by-score" source="films"
  order-by="rt_score:desc">
</dsfr-data-query>

<!-- Films tries par duree decroissante -->
<dsfr-data-query id="by-duration" source="films"
  order-by="running_time:desc">
</dsfr-data-query>

<!-- Nb de films par realisateur -->
<dsfr-data-query id="by-director-count" source="films"
  group-by="director"
  aggregate="id:count"
  order-by="id__count:desc">
</dsfr-data-query>

<!-- Score RT moyen par realisateur -->
<dsfr-data-query id="by-director-score" source="films"
  group-by="director"
  aggregate="rt_score:avg, id:count"
  order-by="rt_score__avg:desc">
</dsfr-data-query>

<!-- Films recents (>= 2000) -->
<dsfr-data-query id="recent-films" source="films"
  where="release_date:gte:2000"
  order-by="release_date:asc">
</dsfr-data-query>

Points cles de cet exemple

  • Provider generic : dsfr-data-source utilise une URL directe vers une API REST qui retourne un tableau JSON plat. Pas besoin d'api-type ni de transform — le cas d'usage le plus simple. L'API Ghibli supporte CORS nativement, donc pas besoin de use-proxy.
  • Normalisation des types : l'API retourne rt_score, running_time et release_date en string. dsfr-data-normalize numeric les convertit en nombres pour permettre le tri, les agregations et les filtres numeriques.
  • Queries multiples sur une seule source : 6 dsfr-data-query derivent chacun une vue differente des memes donnees (tri, group-by, aggregate, where) — tout en client-side.
  • KPI avec agregation inline : dsfr-data-kpi value="rt_score:avg" calcule la moyenne directement, sans query intermediaire.
  • Facettes et tableau interactif : dsfr-data-facets en mode select filtre par realisateur et producteur, dsfr-data-list ajoute la recherche, le tri et l'export CSV.
  • Accessibilite : dsfr-data-a11y sur chaque graphique pour le telechargement CSV.
  • 0 ligne de JavaScript : tout le dashboard est 100% declaratif en HTML.
  • CORS natif : l'API Ghibli autorise les requetes cross-origin — pas besoin de proxy. Pour les APIs sans CORS, utiliser use-proxy pour router via /cors-proxy.
