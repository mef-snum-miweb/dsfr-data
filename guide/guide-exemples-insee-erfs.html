Dashboard revenus - API INSEE

Tableau de bord complet sur les revenus des menages en France, construit a partir de l'API INSEE Melodi avec un pipeline declaratif : source, normalisation progressive (flatten + decodage des codes INSEE), queries et visualisations DSFR.

Architecture du pipeline

dsfr-data-source "raw" (fetch INSEE Melodi, transform="observations") \-- dsfr-data-normalize "n1" (flatten="dimensions") \-- dsfr-data-normalize "n2" (flatten="measures") \-- dsfr-data-normalize "data" (flatten="OBS_VALUE_NIVEAU" + numeric + trim + replace-fields) |-- dsfr-data-query "q-kpi-*" (filtres KPI 2023) | \-- dsfr-data-kpi (4 indicateurs) |-- dsfr-data-query "q-evolution" (serie temporelle) | \-- dsfr-data-chart line |-- dsfr-data-query "q-d1" / "q-d9-evol" (deciles) | \-- dsfr-data-chart line x2 |-- dsfr-data-query "q-pcs" (par PCS) | \-- dsfr-data-chart bar horizontal |-- dsfr-data-query "q-age" (par tranche d'age) | \-- dsfr-data-chart bar |-- dsfr-data-query "q-statut" / "q-menages" | \-- dsfr-data-chart bar + line |-- dsfr-data-query "q-seul" / "q-couple" (type de menage) | \-- dsfr-data-chart line x2 \-- dsfr-data-query "q-table" (donnees detaillees) \-- dsfr-data-list (tableau + export CSV)

L'API INSEE Melodi retourne des observations avec des objets imbriques (dimensions, measures, OBS_VALUE_NIVEAU). Trois dsfr-data-normalize successifs aplatissent ces niveaux, puis replace-fields decode les codes INSEE (PCS, AGE, EMPSTA_ENQ, ERFS_MEASURE) en libelles humains. Les dsfr-data-query filtrent ensuite par indicateur, periode et dimensions pour alimenter chaque visualisation.

Source : INSEE — Enquete Revenus Fiscaux et Sociaux (ERFS), series longues menages
API : api.insee.fr/melodi/data/DS_ERFS_MENAGE_SL
Pipeline 100% declaratif : dsfr-data-source → dsfr-data-normalize ×3 (flatten) + replace-fields (decodage INSEE) → dsfr-data-query → visualisations 

<!-- 1. Fetch API INSEE Melodi -->
<dsfr-data-source id="raw"
  url="https://api.insee.fr/melodi/data/DS_ERFS_MENAGE_SL"
  transform="observations">
</dsfr-data-source>

<!-- 2. Aplatir .dimensions -->
<dsfr-data-normalize id="n1" source="raw" flatten="dimensions"></dsfr-data-normalize>

<!-- 3. Aplatir .measures -->
<dsfr-data-normalize id="n2" source="n1" flatten="measures"></dsfr-data-normalize>

<!-- 4. Aplatir .OBS_VALUE_NIVEAU + forcer numerique + decoder les codes INSEE -->
<dsfr-data-normalize id="data" source="n2"
  flatten="OBS_VALUE_NIVEAU"
  numeric="value"
  trim
  replace-fields="PCS:1T2:Agriculteurs, artisans, commercants
    | PCS:3:Cadres et prof. intel. sup.
    | PCS:4:Professions intermediaires
    | PCS:5:Employes | PCS:6:Ouvriers
    | AGE:Y_LT30:Moins de 30 ans | AGE:Y30T39:30-39 ans
    | AGE:Y40T49:40-49 ans | AGE:Y50T64:50-64 ans
    | AGE:Y65T74:65-74 ans | AGE:Y_GE75:75 ans et plus
    | EMPSTA_ENQ:1T2:Actifs | EMPSTA_ENQ:2:Chomeurs
    | EMPSTA_ENQ:3:Retraites
    | ERFS_MEASURE:MEAN_DI_CONST:Revenu moyen
    | ERFS_MEASURE:MED_DI_CONST:Revenu median
    | ERFS_MEASURE:D1_DI_CONST:1er decile (D1)
    | ERFS_MEASURE:D9_DI_CONST:9e decile (D9)
    | ERFS_MEASURE:NUM_HH:Nombre de menages">
</dsfr-data-normalize>

<!-- KPIs -->
<dsfr-data-query id="q-kpi-mean" source="data"
  where="ERFS_MEASURE:eq:Revenu moyen, PCS:eq:_T, AGE:eq:_T,
    EMPSTA_ENQ:eq:_T, TPH:eq:_T, COMPARE_TIME:eq:NB, TIME_PERIOD:eq:2023">
</dsfr-data-query>

<div class="kpi-row">
  <dsfr-data-kpi source="q-kpi-mean" value="value" label="Revenu disponible moyen"
    format="euro" icon="ri-money-euro-circle-line" color-token="bleu"></dsfr-data-kpi>
  <!-- + D1, Median, D9 (meme pattern) -->
</div>

<!-- Evolution -->
<dsfr-data-query id="q-evolution" source="data"
  where="ERFS_MEASURE:eq:Revenu moyen, PCS:eq:_T, AGE:eq:_T,
    EMPSTA_ENQ:eq:_T, TPH:eq:_T, COMPARE_TIME:eq:NB"
  order-by="TIME_PERIOD:asc">
</dsfr-data-query>

<dsfr-data-chart id="chart-evol" source="q-evolution" type="line"
  label-field="TIME_PERIOD" value-field="value"
  unit-tooltip=" €" selected-palette="default">
</dsfr-data-chart>
<dsfr-data-a11y for="chart-evol" source="q-evolution" table download></dsfr-data-a11y>

<!-- Par PCS (bar horizontal) -->
<dsfr-data-query id="q-pcs" source="data"
  where="ERFS_MEASURE:eq:Revenu moyen, PCS:neq:_T, AGE:eq:18 ans et plus,
    EMPSTA_ENQ:eq:_T, TPH:eq:_T, COMPARE_TIME:eq:NB, TIME_PERIOD:eq:2022"
  order-by="value:desc">
</dsfr-data-query>

<dsfr-data-chart id="chart-pcs" source="q-pcs" type="bar"
  label-field="PCS" value-field="value"
  horizontal unit-tooltip=" €" selected-palette="categorical">
</dsfr-data-chart>

<!-- Par age -->
<dsfr-data-query id="q-age" source="data"
  where="ERFS_MEASURE:eq:Revenu moyen, PCS:eq:_T,
    AGE:in:Moins de 30 ans|30-39 ans|40-49 ans|50-64 ans|65-74 ans|75 ans et plus,
    EMPSTA_ENQ:eq:_T, TPH:eq:_T, COMPARE_TIME:eq:NB, TIME_PERIOD:eq:2023"
  order-by="AGE:asc">
</dsfr-data-query>

<dsfr-data-chart id="chart-age" source="q-age" type="bar"
  label-field="AGE" value-field="value"
  unit-tooltip=" €" selected-palette="sequentialAscending">
</dsfr-data-chart>

<!-- Tableau detaille -->
<dsfr-data-query id="q-table" source="data"
  where="PCS:eq:_T, AGE:eq:_T, EMPSTA_ENQ:eq:_T, TPH:eq:_T, COMPARE_TIME:eq:NB"
  order-by="TIME_PERIOD:desc">
</dsfr-data-query>

<dsfr-data-list source="q-table"
  columns="TIME_PERIOD:Annee, ERFS_MEASURE:Indicateur, value:Valeur"
  search filters="ERFS_MEASURE" sort="TIME_PERIOD:desc"
  pagination="15" export="csv">
</dsfr-data-list>

Points cles de cet exemple

  • 1 seule source HTTP : dsfr-data-source fetch l'API INSEE Melodi, transform="observations" extrait le tableau d'observations.
  • Aplatissement progressif : 3 dsfr-data-normalize successifs aplatissent les objets imbriques (dimensions → measures → OBS_VALUE_NIVEAU).
  • Decodage INSEE via replace-fields : les codes de dimension (PCS 3, AGE Y30T39, etc.) sont convertis en libelles humains directement dans le pipeline, sans JavaScript.
  • Filtres declaratifs : chaque dsfr-data-query filtre par indicateur, periode, PCS, age, etc. avec la syntaxe colon.
  • Visualisations variees : KPIs, courbes d'evolution, barres horizontales, barres verticales, tableaux avec export CSV.
  • Accessibilite : dsfr-data-a11y sur chaque graphique permet le telechargement CSV pour les lecteurs d'ecran.
  • 0 ligne de JavaScript : tout le dashboard est 100% declaratif en HTML.
