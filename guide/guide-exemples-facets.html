dsfr-data-facets

dsfr-data-facets affiche des filtres a facettes interactifs conformes DSFR (checkboxes, selects, multiselects, radios) et filtre les donnees en amont avant de les redistribuer aux composants en aval. Deux modes : client-side (calcul local) et server-facets (API ODS).

Mode server-facets

En mode server-facets, le calcul des facettes est delegue a l'API (ODS, Grist). L'attribut fields est requis car l'auto-detection n'est pas possible. Les compteurs se mettent a jour dynamiquement sur le dataset complet cote serveur.

Tableau filtrable — Rappels de produits avec facettes serveur

Facettes serveur ODS sur Categorie (multiselect) et Sous-categorie (select) avec colonnage DSFR (cols="4"). Les compteurs se mettent a jour dynamiquement sur le dataset complet. Les donnees filtrees alimentent un dsfr-data-list. Tester dans le Playground

Chargement...

<dsfr-data-source id="raw" api-type="opendatasoft"
  dataset-id="rappelconso-v2-gtin-trie"
  base-url="https://data.economie.gouv.fr"
  server-side page-size="20"></dsfr-data-source>

<dsfr-data-facets id="filtered" source="raw"
  server-facets
  fields="categorie_produit, sous_categorie_produit"
  labels="categorie_produit:Categorie | sous_categorie_produit:Sous-categorie"
  display="categorie_produit:multiselect | sous_categorie_produit:select"
  cols="4">
</dsfr-data-facets>

<dsfr-data-list source="filtered"
  columns="modeles_ou_references:Produit, marque_produit:Marque, categorie_produit:Categorie, sous_categorie_produit:Sous-cat."
  server-sort
  pagination="20"
  export="csv">
</dsfr-data-list>

Barres — Beneficiaires filtres par region

La facette Region en mode multiselect filtre les donnees, puis dsfr-data-query agrege pour le graphique en barres. Selectionner des regions recalcule instantanement le graphique. Tester dans le Playground

Chargement...

<dsfr-data-source id="raw" api-type="opendatasoft"
  dataset-id="industrie-du-futur"
  base-url="https://data.economie.gouv.fr"></dsfr-data-source>

<dsfr-data-normalize id="clean" source="raw"
  numeric="montant_investissement, montant_participation_etat, nombre_beneficiaires"
  rename="nom_region:Region | nom_departement:Departement"
  trim>
</dsfr-data-normalize>

<dsfr-data-facets id="filtered" source="clean"
  fields="Region"
  display="Region:multiselect"
  sort="alpha">
</dsfr-data-facets>

<dsfr-data-query id="stats" source="filtered"
  group-by="Departement"
  aggregate="nombre_beneficiaires:sum:Beneficiaires"
  order-by="Beneficiaires:desc"
  limit="15">
</dsfr-data-query>

<dsfr-data-chart source="stats"
  type="bar"
  label-field="Departement"
  value-field="Beneficiaires"
  selected-palette="categorical">
</dsfr-data-chart>

Modes d'affichage

L'attribut display permet de choisir le mode d'affichage par facette. Quatre modes sont disponibles :

  • checkbox (defaut) : checkboxes DSFR dans un fieldset ouvert
  • select : liste deroulante native DSFR, selection exclusive
  • multiselect : dropdown DSFR avec checkboxes, recherche et bouton "Tout selectionner/deselectionner"
  • radio : dropdown DSFR avec radio buttons, recherche, selection exclusive

Les 4 modes — radio, multiselect, select, checkbox

Chaque facette utilise un mode different. L'attribut hide-counts masque les compteurs.

Chargement...

<dsfr-data-source id="raw" api-type="opendatasoft"
  dataset-id="rappelconso-v2-gtin-trie"
  base-url="https://data.economie.gouv.fr"
  server-side page-size="20"></dsfr-data-source>

<dsfr-data-facets id="filtered" source="raw"
  server-facets
  fields="categorie_produit, sous_categorie_produit, marque_produit"
  labels="categorie_produit:Categorie | sous_categorie_produit:Sous-categorie | marque_produit:Marque"
  display="categorie_produit:radio | sous_categorie_produit:multiselect | marque_produit:select"
  max-values="6">
</dsfr-data-facets>

<dsfr-data-list source="filtered"
  columns="modeles_ou_references:Produit, marque_produit:Marque, categorie_produit:Categorie"
  server-sort
  pagination="20">
</dsfr-data-list>

Mode server-facets (ODS)

En mode server-facets, dsfr-data-facets fetche les valeurs de facettes depuis l'API ODS /facets au lieu de les calculer localement. Cela permet des facettes dynamiques sur des datasets de millions de records. Les compteurs se mettent a jour automatiquement selon la recherche et les selections cross-facet.

Requis : source pointant vers un dsfr-data-source avec api-type="opendatasoft" et server-side. L'attribut fields est obligatoire (pas d'auto-detection en mode serveur).

Recherche + facettes serveur + cartes

Pipeline complet server-side : dsfr-data-search server-search et dsfr-data-facets server-facets envoient leurs filtres a dsfr-data-source qui merge les where avec AND. Les compteurs de facettes se mettent a jour dynamiquement (cross-facet). Tester dans le Playground

Chargement...

<dsfr-data-source id="q" api-type="opendatasoft"
  dataset-id="industrie-du-futur"
  base-url="https://data.economie.gouv.fr"
  server-side page-size="12">
</dsfr-data-source>

<dsfr-data-search source="q" server-search
  placeholder="Rechercher une entreprise..."
  count>
</dsfr-data-search>

<dsfr-data-facets id="filtered" source="q"
  server-facets
  fields="nom_region,type_aide"
  labels="nom_region:Region | type_aide:Type d'aide"
  display="nom_region:multiselect | type_aide:select"
  cols="6">
</dsfr-data-facets>

<dsfr-data-display source="filtered" cols="3" pagination="12">
  <template>
    <div class="fr-card fr-card--shadow">
      <div class="fr-card__body">
        <div class="fr-card__content">
          <h3 class="fr-card__title">{{nom_entreprise}}</h3>
          <p class="fr-card__desc">{{nom_departement}} — {{nom_region}}</p>
          <div class="fr-card__start">
            <p class="fr-badge fr-badge--sm fr-badge--green-emeraude">{{type_aide}}</p>
          </div>
        </div>
      </div>
    </div>
  </template>
</dsfr-data-display>

Attributs

Attribut Type Description
sourceStringID de la source de donnees
fieldsStringChamps a utiliser comme facettes (CSV). Si vide, auto-detection en mode client
labelsStringLabels personnalises : champ:Label | champ2:Label2
max-valuesNumberNombre max de valeurs affichees par facette (defaut : toutes)
disjunctiveStringChamps en mode disjonctif (OR au lieu de AND) : CSV
sortStringTri des valeurs : count (defaut), alpha, alpha-desc
searchableStringChamps avec barre de recherche dans les facettes : CSV
hide-emptyBooleanMasquer les groupes avec 0 ou 1 valeur
displayStringMode d'affichage : checkbox (defaut), select, multiselect, radio
hide-countsBooleanMasquer les compteurs a cote de chaque valeur de facette
url-paramsBooleanInitialise les selections depuis les parametres URL
url-param-mapStringMapping URL param → champ : param:field | param2:field2
url-syncBooleanSynchronise l'URL quand l'utilisateur change les facettes
colsStringColonnage DSFR : "4" (global) ou "champ:4 | champ2:6" (par facette). Utilise la grille DSFR fr-col-N
server-facetsBooleanActive le mode facettes serveur ODS (API /facets). Requiert source vers un dsfr-data-source api-type="opendatasoft" server-side (directement ou via un dsfr-data-query, qui relaie automatiquement les commandes). fields obligatoire
